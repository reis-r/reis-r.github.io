Three months without Emacs: do I miss it?

Published: 04/10/2020
Last edited: 04/10/2020
By Rony Reis
Category: misc
Tags: emacs unix productivity

I used to do it all in Emacs, checking my email, chatting, reading news, planning tasks and even listening to internet radio and podcasts. I used spit a thousand commands per second with the amazing xah-fly-keys modal editing. I was very productive indeed, I didn't even launch X if I didn't needed to. In fact, I had a Nix script that built Emacs and it's packages together with my system stuff, it was basically Nix/Emacs! hehehe

Then, I narrowed my studies to Python and Data Science, and a graphical interface turned into something just so essential, then came Jupyter, and I bought a new computer which came with Windows... I felt I could try something really different to make my life more sane (because I try to do so many different things in life), and rather than just changing my Window Manager, I re-evaluated my whole toolset.

For most production stuff I used Emacs for, like to-do lists and calendar, I'm using Outlook now, synchronizing with my phone is always good. In fact, most non-programming things I did before now are done faster and more nicely (UX-wise) with office tools.

For Notebooks (my work tool as a data guy), EIN is good. But I got really used to Jupyter Lab, it's way easier to configure. I also like to use the Google Colab platform when I need to show someone the code (it makes a lot easier to share). In the R side, RStudio is just too good to ignore, it's like "Excel for serious math". Ok, this is like 4 different interfaces to learn, but once you get good at them, I think production goes up so much, and most of them are way easier to learn than Emacs. Maybe I'm just experiencing the universal facts of QWERTY Theory. I feel a lot more productive just not trying too hard.

The only thing I really miss Emacs is actually coding (this sounds silly!). I have been using the Atom editor, which looks like a good Emacs replacement, but I really could not dig too much into it. (Help! I don't see a RELP!) I miss coding modes for more hipster languages like the Hy Programming Language, which I love. I'll probably install Emacs soon to play with this language again, I just need to decide: MSYS, Cygwin, WSL... Maybe native?
